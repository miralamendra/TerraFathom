# UX LAWS

## Principle

The interface should require zero explanation.

Users should understand what to do by looking.

---

# Attention Flow

Eye path:

Top Left

Toolbar

Datasets

Layers

Map

Properties

Bottom Drawer

Never interrupt this flow.

---

# Cognitive Load

Every visible element costs attention.

Remove anything unnecessary.

If an element does not help:

* understanding
* navigation
* action

remove it.

---

# Discoverability

Primary actions must always be visible.

Examples:

Add Data

Create Layer

Filter Data

Map Style

Never hide primary actions.

---

# Layer Workflow

User journey:

Load Dataset

↓

Create Layer

↓

Configure Layer

↓

Analyze

The UI should visually support this flow.

---

# Dataset Workflow

Upload area should be strongest element.

Sample datasets should be secondary.

Samples are helpers.

Not features.

Never highlight samples more than upload.

---

# Properties Workflow

Most common settings first.

Order:

Visibility

Color

Radius

Opacity

Stroke

Elevation

Data Mapping

Advanced

Never reverse.

---

# Interaction Feedback

Hover

Subtle

Pressed

Immediate

Focused

Obvious

Disabled

Clearly disabled

Nothing should feel uncertain.

---

# Forms

Every slider must have numeric input.

Never slider only.

Every value must be editable.

Example:

Radius

[ slider ]

[ 24 px ]

Opacity

[ slider ]

[ 70 % ]

---

# Professional Tool Rule

This is desktop software.

Not a website.

Not a dashboard.

Not a marketing page.

Think:

Figma

VS Code

ArcGIS Pro

Blender

Linear

Information density is good.

Visual noise is bad.
