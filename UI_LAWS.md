# UI LAWS

## Philosophy

Reduce before adding.

Simplify before decorating.

Use whitespace before borders.

Use typography before color.

Use alignment before decoration.

---

# Visual Hierarchy

Priority order:

1. Map
2. Active Layer
3. Dataset
4. Primary Action
5. Secondary Action
6. Metadata
7. Decorative Elements

Anything lower must never compete with anything above.

---

# Surface Law

Only 3 surface levels allowed.

Level 0

Application background

Level 1

Toolbar

Sidebar

Properties panel

Bottom drawer

Level 2

Inputs

Dropdowns

Buttons

Menus

Nothing else.

No extra cards.

No nested surfaces.

---

# Rectangle Law

Rectangles are expensive.

Never create a rectangle unless interaction requires it.

Allowed:

* buttons
* inputs
* dropdowns
* dialogs
* menus

Avoid:

* cards
* panels inside panels
* boxes inside boxes

Group using:

* spacing
* alignment
* typography

Not borders.

---

# Border Law

Use borders only for separation.

Allowed:

* toolbar divider
* sidebar divider
* properties divider
* drawer divider
* inputs
* dropdowns

Remove all decorative borders.

Never stack borders.

Never border inside border inside border.

# Alignment

Everything aligns to invisible columns.

All left edges align.

All inputs align.

All labels align.

All buttons have equal heights.

No random widths.

No visual drift.

---

# Color Law

Accent color exists only for actions.

Accent is never structural.

Blue = action

Grey = structure

White = information

Muted = metadata

Never use color to create hierarchy.

Use typography and spacing first.

---

# Icons

Icons support text.

Icons never dominate text.

Default:

14px

Maximum:

16px

Decorative icons prohibited.

Every icon must have purpose.

---

# Empty States

Must feel premium.

Never childish.

Never illustration-heavy.

Simple.

Professional.

Actionable.
